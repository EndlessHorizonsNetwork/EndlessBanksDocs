Setup e dipendenza

Per utilizzare l'API di EndlessBanks nel tuo plugin è sufficiente aggiungere EndlessBanksAPI come dipendenza Maven con scope provided. Il jar verrà risolto dal tuo repository locale (.m2), previo install.

1. Installa l'API nel repository locale

Richiedi il file EndlessBanksAPI-1.0.jar allo sviluppatore principale ed esegui:

# Da terminale, nella directory dove si trova il jar
mvn install:install-file \
  -Dfile=EndlessBanksAPI-1.0.1.jar \
  -DgroupId=it.endlesshorizons \
  -DartifactId=EndlessBanksAPI \
  -Dversion=1.0 \
  -Dpackaging=jarBASH

2. Aggiungi la dipendenza al pom.xml

Per Maven:

<dependency>
    <groupId>it.endlesshorizons</groupId>
    <artifactId>EndlessBanksAPI</artifactId>
    <version>1.0.1</version>
    <scope>provided</scope>
</dependency>XML

Per Gradle:

repositories {
    mavenLocal()
}

dependencies {
    compileOnly('it.endlesshorizons:EndlessBanksAPI:1.0.1')
}GRADLE

3. Dichiara la dipendenza nel plugin.yml

EndlessBanks deve essere caricato prima del tuo plugin. Aggiungilo come dipendenza obbligatoria:

name: TuoPlugin
version: 1.0
main: it.tuonamespace.tuoplugin.TuoPlugin
api-version: '1.21'
depend: [EndlessBanks]   # dipendenzaYAML
🚀

Getting started

L'intero accesso all'API avviene tramite il singleton EndlessBanksAPI.get(), disponibile dopo che EndlessBanks ha completato il proprio onEnable(). Poiché hai dichiarato depend: [EndlessBanks], quando il tuo plugin viene abilitato EndlessBanks è già operativo. 

import it.endlesshorizons.endlessbanks.api.EndlessBanksAPI;
import it.endlesshorizons.endlessbanks.api.IEndlessBanksAPI;

public class TuoPlugin extends JavaPlugin {

    private IEndlessBanksAPI banksApi;

    @Override
    public void onEnable() {
        // EndlessBanks è già caricato, l'istanza è quindi disponibile
        this.banksApi = EndlessBanksAPI.get();
    }
}JAVA
⚠️ Non chiamare EndlessBanksAPI.get() in fase di caricamento statico della classe o prima di onEnable(). Se EndlessBanks non è caricato, verrà lanciata una IllegalStateException.
💶

ICardManager

Gestisce i conti bancari personali. Ogni utente può avere al massimo una BankCard attiva, associata al proprio UUID. Ottenibile tramite EndlessBanksAPI.get().getCardManager().

ICardManager interface
Metodo Ritorna Descrizione
getCardFrom(UUID ownerUuid) Optional<BankCard> Restituisce la BankCard dell'utente dato il suo UUID. Vuoto se non ha un conto.
getCard(UUID cardUuid) Optional<BankCard> Ricerca una BankCard tramite l'UUID fisico della carta (non del proprietario).
hasAccount(Player player) boolean Verifica se il giocatore ha un conto bancario attivo.
isCard(ItemStack item) boolean Verifica se l'ItemStack è una carta bancaria (attiva o non ancora attivata).
isDeactivatedCard(ItemStack item) boolean Verifica se la carta è presente fisicamente nell'inventario ma non ancora attivata in banca.
getAll() Collection<BankCard> Restituisce tutte le BankCard attualmente in cache (tutti i conti attivi sul server).

Esempio

ICardManager cardManager = EndlessBanksAPI.get().getCardManager();

// Controlla se il giocatore ha un conto
if (!cardManager.hasAccount(player)) {
    player.sendMessage("Devi prima aprire un conto in banca.");
    return;
}

// Ottieni la carta e il saldo
cardManager.getCardFrom(player.getUniqueId()).ifPresent(card -> {
    double saldo = card.getBalance(economy);
    player.sendMessage("Il tuo saldo è: " + saldo + "€");
});JAVA

IBusinessManager

Gestisce i conti aziendali (partite IVA, carte aziendali, tipi di capitale). Un utente può essere titolare di più aziende. Ottenibile tramite EndlessBanksAPI.get().getBusinessManager().

IBusinessManager interface
Metodo Ritorna Descrizione
getCompany(String vatNumber) Optional<BusinessAccount> Ricerca un'azienda tramite la sua Partita IVA (identificatore univoco).
getAccountFrom(String companyName) Optional<BusinessAccount> Ricerca un'azienda per nome (case-insensitive).
getFrom(UUID ownerUuid) List<BusinessAccount> Restituisce tutte le aziende intestate all'utente specificato.
getPersonalBusiness(UUID ownerUuid) Optional<BusinessAccount> Restituisce la ditta individuale (regime forfettario) dell'utente, se presente.
exists(String vatNumber) boolean Verifica se esiste un'azienda con questa Partita IVA.
companyExists(String companyName) boolean Verifica se esiste un'azienda con questo nome.
hasCard(BusinessAccount account) boolean Verifica se l'azienda ha una carta aziendale emessa.
getCard(String vatNumber) BusinessCard Restituisce la carta aziendale associata alla Partita IVA. Può essere null se non emessa.
getBusinessCard(UUID cardUuid) Optional<BusinessCard> Ricerca una BusinessCard tramite il suo UUID fisico.
isBusinessCard(ItemStack item) boolean Verifica se l'ItemStack è una carta aziendale valida (attiva).
isDeactivatedCard(ItemStack item) boolean Verifica se l'ItemStack è una carta aziendale non ancora attivata.
⚠️ I metodi register(BusinessAccount) e register(BusinessAccount, CardModel) creano e persistono dati direttamente nel database. Utilizzarli solo in contesti controllati, la creazione standard avviene attraverso le GUI della Banca e dell'Agenzia delle Entrate, del plugin originario.

BankCard

Rappresenta il conto bancario personale di un utente. Il saldo è gestito integralmente da Vault.

BankCard class
  • StringownerUuid— UUID dell'utente proprietario
  • StringcardUuid— UUID univoco della carta fisica
  • Stringtype— Tipo di carta (es. "standard", "gold")
  • Stringpin— PIN della carta (5 cifre)
  • doubledailyWithdrawn— Importo prelevato oggi (si azzera al cambio giornata)
  • longlastWithdrawTimestamp— Timestamp Unix dell'ultimo prelievo
Metodo Ritorna Descrizione
getBalance(Economy economy) double Saldo Vault del proprietario. Richiede l'istanza di Economy (da Vault).
getRealOwnerUuid() UUID Restituisce l'UUID del proprietario come oggetto UUID tipizzato.
🏛️

BusinessAccount

Rappresenta il conto aziendale (l'anagrafica dell'azienda). Non contiene direttamente il saldo, quello è nella BusinessCard indicato successivamente.

BusinessAccount class
  • StringvatNumber— Partita IVA (11 cifre, identificatore univoco)
  • StringcompanyName— Ragione sociale dell'azienda
  • StringownerUuid— UUID del giocatore titolare
  • CapitalTypecapitalType— Forma giuridica (FLAT_RATE, SRL, SPA)
💼

BusinessCard

Rappresenta la carta aziendale. A differenza della BankCard personale, il saldo è memorizzato internamente (non dipende da Vault).

BusinessCard class
  • StringcardUuid— UUID univoco della carta
  • StringvatNumber— Partita IVA dell'azienda associata
  • StringownerUuid— UUID del titolare
  • Stringtype— Tipo di carta aziendale
  • doublebalance— Saldo della carta aziendale (in €)
  • doubledailyWithdrawn— Prelevato oggi
Metodo Ritorna Descrizione
getBalance() double Saldo attuale della carta aziendale.
add(double amount) void Aggiunge importo al saldo. Non persistere manualmente — il plugin lo gestisce internamente.
subtract(double amount) void Sottrae importo dal saldo.
⚠️ Non chiamare add() o subtract() direttamente da plugin esterni. La modifica del saldo aziendale deve passare dai sistemi interni di EndlessBanks per garantire la registrazione delle transazioni in maniera corretta.
🗂️

CardModel

Descrive il modello di una carta (tipo, limiti, costi). Necessario per la creazione di nuove carte tramite IBusinessManager.register(). I modelli disponibili sono configurati nel server — non crearli manualmente.

CardModel class
  • Stringtype— Identificatore interno del tipo
  • StringtypeDisplayName— Nome visualizzato, supporta anche i colori
  • doublemaxBalance— Saldo massimo consentito
  • doubledailyWithdrawLimit— Limite prelievo giornaliero
  • doublemaxDebt— Debito massimo consentito
  • doubledepositFee— Commissione su deposito
  • doubletransferFee— Commissione su trasferimento
  • booleanallowDirectPayment— Abilita pagamenti diretti via POS

Enumeratori

CapitalType enum

Forma giuridica dell'azienda.

  • FLAT_RATE— Regime forfettario (ditta individuale)
  • SRL— Società a responsabilità limitata
  • SPA— Società per azioni
Metodo Ritorna Descrizione
getNext() CapitalType Restituisce il tipo successivo nella gerarchia (FLAT_RATE → SRL → SPA → null).
TransactionDirection enum
  • IN— Entrata di denaro
  • OUT— Uscita di denaro

Eventi transazioni

EndlessBanks pubblica eventi Bukkit ad ogni transazione registrata. È possibile ascoltarli tramite un Listener standard per reagire a pagamenti, prelievi o movimenti aziendali.

CommonTransactionEvent event

Scattato ad ogni transazione personale registrata.

Metodo Ritorna Descrizione
getTransaction() CommonTransaction Restituisce i dati completi della transazione.
BusinessTransactionEvent event

Scattato ad ogni transazione aziendale registrata (entrate e uscite).

Metodo Ritorna Descrizione
getTransaction() BusinessTransaction Restituisce i dati completi della transazione aziendale.

Esempio listener

public class BankListener implements Listener {

    @EventHandler
    public void onPersonalTransaction(CommonTransactionEvent event) {
        CommonTransaction tx = event.getTransaction();

        if (tx.getDirection() == TransactionDirection.IN) {
            // L'utente ha ricevuto denaro
            Player player = Bukkit.getPlayer(UUID.fromString(tx.getPlayerUuid()));
            if (player != null) {
                player.sendMessage("Hai ricevuto " + tx.getAmount() + "€ da " + tx.getTargetName());
            }
        }
    }

    @EventHandler
    public void onBusinessTransaction(BusinessTransactionEvent event) {
        BusinessTransaction tx = event.getTransaction();
        // Log di tutte le transazioni aziendali
        Bukkit.getLogger().info("[Bank] " + tx.getCompanyName()
            + " | " + tx.getDirection()
            + " | " + tx.getAmount() + "€");
    }
}JAVA

Casi d'uso pratici

Verifica se un utente ha completato l'apertura del conto (uso tipico nel tutorial)

IEndlessBanksAPI api = EndlessBanksAPI.get();

public boolean hasCompleted(Player player) {
    return api.getCardManager().hasAccount(player);
}JAVA

Ottenere il saldo dell'utente

public double getBalance(Player player, Economy economy) {
    return EndlessBanksAPI.get()
        .getCardManager()
        .getCardFrom(player.getUniqueId())
        .map(card -> card.getBalance(economy))
        .orElse(0.0);
}JAVA

Verificare se un utente ha un'azienda attiva con carta emessa

IBusinessManager bm = EndlessBanksAPI.get().getBusinessManager();

public boolean hasCompanyCard(Player player) {
    return bm.getFrom(player.getUniqueId())
             .stream()
             .anyMatch(bm::hasCard);
}JAVA

Reagire ad un pagamento ricevuto tramite POS

@EventHandler
public void onPaymentReceived(BusinessTransactionEvent event) {
    BusinessTransaction tx = event.getTransaction();

    // Interesse solo alle entrate dell'azienda "Negozio Roma"
    if (!tx.getCompanyName().equalsIgnoreCase("Negozio Roma")) return;
    if (tx.getDirection() != TransactionDirection.IN) return;

    // Avvia logica personalizzata (es. avanzamento tutorial, achievement, ecc.)
    onSaleCompleted(tx.getAmount());
}JAVA
✅ Tutti i metodi che restituiscono Optional non lanciano mai eccezioni per dati mancanti. Usa sempre ifPresent(), orElse() o ifPresentOrElse() per gestirli in sicurezza.
